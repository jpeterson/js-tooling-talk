Writing Code

Now I'm going to briefly set up the demo by pinpointing some workflow problems solved by tools

Writing code - is there a "right" JS editor?

Yes and no. Find the editor that does what you need, keep in mind some were made for web development

Visual Studio and Eclipse are okay... but theres a lot of overhead and junk that you'll never use

Sublime Text / Atom / WebStorm


These are examples of modern code editors made for web developers

Sublime Text is my weapon of choice, and Atom is a really hyped editor coming from the GitHub team. WebStorm is more along the lines of a traditional IDE.

Plugins! Backbone and Bootstrap snippets and autocomplete, other stuff we'll see in a few minutes

Syntax Errors

Since JS doesn't compile until runtime, its hard to identify "pre-compilation" errors

Syntax errors, undefined variables, typos

JSHint / JSLint


Linters are programs that identify common errors and potential problems

Dependency Management

For smaller JS projects you may not need dependency management

But as your projects grow in size, you'll be lost without it

Bower


CSS is not code


To developers, CSS can be a bit of a curveball

It's not code, it can be very repetetive and theres a lot of typing and bloat

Stylus / SASS / LESS


To fix this, these things called CSS Preprocessors have started gaining traction

They allow us to write "code" that gets compiled into plain CSS

Task Automation

Now that we know all these great things we can and should be doing, how do we do them efficiently?

Did we just add a ton of effort to our workflow with each of these tools?

Grunt / Gulp / CodeKit / Mixture


If we use one of these guys, they will carry out the repeatable tasks for us

Grunt/Gulp are command line based, while Mixture and CodeKit take a GUI approach

Debugging

Debugging is probably the #1 complaint I hear about JS development

Firebug blazed this trail, but these days Chrome Dev Tools is king

Postman is a relative newcomer that helps with API testing, we might not have time to look at it but you should definitely check it out
